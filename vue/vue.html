    # 事件修饰符
    • 在事件处理程序中调用 event.preventDefault() 或 event.stopPropagation() 是非常常见的需求。
    • Vue 不推荐我们操作DOM 为了解决这个问题，Vue.js 为 v-on 提供了事件修饰符
    • 修饰符是由点开头的指令后缀来表示的
    <!-- 阻止单击事件继续传播 -->
<a v-on:click.stop="doThis"></a>

<!-- 提交事件不再重载页面 -->
<form v-on:submit.prevent="onSubmit"></form>

<!-- 修饰符可以串联   即阻止冒泡也阻止默认事件 -->
<a v-on:click.stop.prevent="doThat"></a>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThat">...</div>

使用修饰符时，顺序很重要；相应的代码会以同样的顺序产生。因此，用 v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。

    # 5.计算属性

    # 解决问题

    在插值中使用表达式,可读性差，不利于维护

    通过get() 获取相关属性的变化计算自己，通过set()监视自己的属性的主动变化(因此可以用来影响关联属性)

    # 作用

    ​ 通过关联属性对原始属性进行变更

    # 用法

    ​ Vue选项computed:{}

    计算属性是基于它们的响应式依赖进行缓存的,只在相关响应式依赖发生改变时它们才会重新求值。

    计算属性必须返回一个结果（计算过后的结果）

    计算属性作为函数出现时默认的是：

    ​ 这个属性对象的getter

    计算属性对象的setter其实是一个监视器，只要计算属性的本体发生变化,就会触发

    # 应用场景

    ​ 面向不同国家的一个商城，里面的价格只有一份，不同国家的人看到的货币都是经过汇率换算过的结果，这时就可以使用到计算属性进行计算

    PS:单选项中访问其他选项的属性或方法都需要通过this指向vm实例再通过实例得到其他选项中的属性或方法

    <div id="app">
    第一个输入框 <input v-model="input1"> <br>
    第二个输入框 <input v-model="input2"> <br>
    前两个数字相加的结果 <input v-model="result"> <br>
</div>
</body>
</html>
<script>
    const app = new Vue({
        el:"#app",
        computed:{
            result:{
                //这其实是result属性的回调，回调的时机是获取当前属性值的时候，作用是将返回值作为属性值
                get(){
                    let regNum = /^\d*$/ //申明数字的正则表达式
                    let num1 = parseFloat(this.input1)
                    let num2 = parseFloat(this.input2)
                    if (regNum.test(num1) && regNum.test(num2)){
                        return num1 + num2
                    }else{
                        return this.input1 + this.input2
                    }
                },
                //这也是一个回调，回调的实际是属性值发生改变的时候，监视当前属性值的变化，同步更新相关属性值
                set(value){
                    let regNum = /^\d*$/ //申明数字的正则表达式
                    let num1 = parseFloat(this.input1)
                    let num2 = parseFloat(this.input2)
                    //如果是数字类型，就转化后相加
                    if (regNum.test(num1) && regNum.test(num2)){
                        this.input1 = value - this.input2
                        this.input2 = value - this.input1
                        value = num1 + num2
                        //如果不是数字类型就拆分字符串
                    }else{
                        this.input1= value.split(this.input2)[0]
                        this.input2= value.split(this.input1)[1]
                        value = this.input1 + this.input2
                    }
                }
            }
        },
        data:{
            input1:"",
            input2:""
        }

    })
</script>

    # 6.侦听器

    观察和响应 Vue 实例上的数据变动,计算属性也是如此

    # 侦听器的参数

    参数1：更改后的值,参数2:更改前的值

    # 侦听器与计算属性的区别

    侦听器侦听对象的本身，并且在其他的选项中已经存在而计算属性不用

    1. 侦听器，侦听自己，当值发生变化的时候会触发，有两个参数（新值，旧值）
    2. 计算属性，侦听关联属性，当关联属性的值发生变化的时候，就得到一个值
    3. 计算属性具有缓存，证明方式是多次调用同一个计算属性，观察执行多少次，侦听器是每次变化都会执行
    4. 每个计算属性都有两个回调函数，一个是setter，一个是getter
    5. get用来返回当前属性的值，不会影响原始属性的值
    6. set用于监视属性，一旦属性改变，就会调用set，就会将新的值传入set(value)

    //注册全局事件中心
    let parent = new Vue();

    //组件A
    Vue.component("azujian",{
        data(){
            return {
                num:0
            }
        },
        template:`<div>
            A组件的计数器读数:{{num}} <br>
            <button @click="adomchange">操作B组件的计数器读数</button>

            adomchange(){

        mounted(){
            //启用事件中心的监听,当bzujian被绑定了aar的那个事件被触发的时候,就调用这个方法
            parent.$on('aar',(num)=>{
                this.num += num

    //组件B
    Vue.component("bzujian",{
        props:["counter"],
        template:`<div>
            B组件的计数器读数:{{num}} <br>
            <button @click="bdomchange">操作A组件的计数器读数</button>
        </div>`,
        data(){
            return {
                num:0
            }
        },
        methods:{
            //写本组件的方法

    # 8-1.路由概述

    官方原文：

    用 Vue.js + Vue Router 创建单页应用，感觉很自然：使用 Vue.js ，我们已经可以通过组合组件来组成应用程序，当你要把 Vue Router 添加进来，我们需要做的是，将组件 (components) 映射到路由 (routes)，然后告诉 Vue Router 在哪里渲染它们。

    要解决的问题：更加方便的创建单页面应用

    创建方式： 1引入vue.js >>> 2引入VueRouter>>>>3创建路由组件>>>>4创建VueRouter实例>>>在VueRouter里面

    # 前后端路由的差别：

    后端路由： 根据不同的URL请求，返回不同的服务器资源---增加服务器负荷，导致性能问题 前端路由： 根据不同的用户事件，显示不同的页面内容(显示的是不同的组件，路由的本质就是把不同的组件回显给不同的用户事件) 采用前端路由，可以解决ajax技术导致的浏览器不支持前进后退的问题，实现单页面应用程序(SPA)，也就是整个网站只有一个页面，不过这种技术不利于提高SEO，不利于商业推广

    # 路由的渲染原理：-

    vue-router.js渲染的原理 1、router-link 这个标签会被渲染为a标签 2、router-link 的to属性会被渲染为href属性 3、router-link 的to属性的值会被渲染为herf属性的值，就是以#开头的hash地址

      next: Function: 一定要调用该方法来 resolve 这个钩子。执行效果依赖 next 方法的调用参数。
    * */
    //如果访问的是登陆页面,就直接放行
    if (to.path === "/login") {
        return next()
    }
    //如果要访问其他页面,就先获取token,如果token不为空,就放行,否则就强制跳转到登录页面
    const token = window.sessionStorage.getItem('token');
    //判断token是否为空
    if (!token) {
        // Message.error("你没有登陆,无权访问本页面,已为你跳转到登陆页面")
        return next("login")
    } else {
        //dochain
        return next()
    }
})
